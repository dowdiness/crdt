# §P0b Phase 1 Workspace Coordinator — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the §P0b Phase 1 workspace coordinator: a shared-`@incr.Runtime` Coordinator that mediates Lambda editor construction, protected-cell reads, and lifecycle through a typed checked-read API, replacing today's fresh-per-editor Runtime + duplicated FFI ctors.

**Architecture:** Four PRs in dependency order.
1. **Loom A+B (one submodule PR)** — add `Runtime::gc_root_count` introspection; mechanical `Memo → Derived` rename in `loom/examples/lambda/{typecheck,callers}`.
2. **Canopy 1 (canopy PR)** — `VersionedFlatProj` switches `BackdateEq`/`HasChangedAt` for a custom `Eq`; `proj_memo` + `typed_term_memo` migrate `Memo → Derived`; 6 new `pub` accessors on `SyncEditor`; `new_generic` gains `parent_runtime?` pass-through.
3. **Canopy 2 (canopy PR)** — new `workspace/coordinator/` package; `LambdaProtectedCells` + `assemble_lambda_handle` helper; FFI ctors funnel through the helper.
4. **(out-of-scope, listed for context only)** Phase 1b ports production accessors to `Coordinator::read_protected` and lands the first workspace memo via `register_dep`.

**Tech Stack:** MoonBit, `dowdiness/incr` (target API: `Input`/`Derived`/`Watch`), `dowdiness/loom` (parser), canopy `editor` + `lang/lambda` packages, FFI lifecycle.

**Frozen design references:**

- Spec: `docs/superpowers/specs/2026-05-24-p0b-phase1-skeleton-design.md` (branch `docs/p0b-call-flow-grounding`, commits `dfdb994` + `3081484`). Cited below as `spec §N`.
- Grounding: `docs/research/2026-05-24-shared-runtime-call-flow-grounding.md` (same branch).
- §13 decision adoption (resolved before plan-writing):
  - D1 Loom A + Loom B bundled into one loom PR.
  - D2 `unregister_dep` is `pub` from day one.
  - D3 `AbortReport.domain_tag` is free-form prose in Phase 1.

  - D5 Test 5 (`ProtectedCellDisposed`) is a whitebox test, not a `pub dispose` on `ProtectedCell`.

  ///|
  pub fn Coordinator::register_dep(
    self : Coordinator,
    workspace_memo : @incr.CellId,
    editor_id : EditorId,
    editor_cell : @incr.CellId,
  ) -> Unit {
    let entry = match self.deps.get(workspace_memo) {
      Some(arr) => arr
      None => {
        let arr : Array[(EditorId, @incr.CellId)] = []
        self.deps.set(workspace_memo, arr)
        arr
      }
    }
    let edge = (editor_id, editor_cell)
    if !entry.contains(edge) {
      entry.push(edge)
    }
  }

  ///|
  /// Idempotent. Phase 1 callers: the destroy-gateway test (PR4). Phase 1b+:
  /// workspace memos released on their own dispose.
  pub fn Coordinator::unregister_dep(
    self : Coordinator,
    workspace_memo : @incr.CellId,
    editor_id : EditorId,
    editor_cell : @incr.CellId,
  ) -> Unit {
    let entry = match self.deps.get(workspace_memo) {
      Some(e) => e
      None => return
    }
    let edge = (editor_id, editor_cell)
    // Linear scan + remove-by-value. Phase 1 has at most a handful of
    // edges per workspace memo; profile in Phase 2 if this becomes hot.
    let new_entry = entry.filter(fn(e) { e != edge })
    if new_entry.is_empty() {
      self.deps.remove(workspace_memo)
    } else {
      self.deps.set(workspace_memo, new_entry)
    }
  }
  ```

  > **Note:** `Array::remove(value)` is not in MoonBit core. The implementation above uses `Array::filter` to construct a fresh array, then writes it back. If a future incr/canopy version exposes mutation-in-place by value, the optimization is local.

  ///|
  pub fn Coordinator::destroy_editor(
    self : Coordinator,
    editor_id : EditorId,
  ) -> Result[Unit, AbortReport] {
    let reg = match self.editors.get(editor_id.0) {
      Some(r) => r
      None =>
        return Err(
          AbortReport(
            kind=EditorDestroyed,
            editor_id~,
            agent_id="<unknown>",
          ),
        )
    }
    guard reg.alive.val else {
      return Err(
        AbortReport(
          kind=EditorDestroyed,
          editor_id~,
          agent_id=reg.agent_id,
        ),
      )
    }
    let referring : Array[@incr.CellId] = []
    for ws_memo, edges in self.deps {
      for edge in edges {
        if edge.0 == editor_id {
          referring.push(ws_memo)
        }
      }
    }
    if referring.length() > 0 {
      return Err(
        AbortReport(
          kind=DestroyWhileDependedUpon,
          editor_id~,
          agent_id=reg.agent_id,
          cell_id=Some(referring[0]),
        ),
      )
    }
    // Flip alive=false BEFORE disposing watches (spec §7 lines 420-425).
    // Hardens against re-entrant reads during disposal.
    reg.alive.val = false
    for p in reg.protected {
      (p.dispose)()
    }
    Ok(())
  }
  ```

- [ ] **Step 4: Write test 6 — CycleDetected mapping (hand-built ProtectedCell)**

  This test verifies the `read_protected → AbortKind::CycleDetected` *mapping*, not loom's cycle-detection algorithm (which is already covered by incr's own test suite).       cell_id=d.id(),

      dispose=fn() { ignore(0) },

## Spec gaps surfaced during plan-writing

These were discovered while drafting executable steps and flagged here per CLAUDE.md "Separating Design from Implementation." The plan adopts a workaround for each; the user should decide whether to amend the spec or accept the narrowing.

1. **Spec §12 test 6 — real-cycle reproduction is unreliable.** The spec phrases test 6 as "Construct a synthetic Derived chain with a cycle; register; read; expect `Err(AbortReport { kind: CycleDetected, ... })`." However, a Derived whose compute closure calls `read_or_abort()` on a self-reference aborts at cycle detection inside the closure, and `loom/incr/cells/memo.mbt:222` wraps that abort with `catch { e => abort("Memo compute raised: " + e.to_string()) }`. The outer `Watch::read` therefore receives a hard abort, not `Err(CycleError)`, and the test never reaches the coordinator's mapping branch. **Plan workaround:** test 6 is reframed as a unit-style mapping test that hand-builds a `ProtectedCell` with a `read` closure returning `Err(CycleError)` directly. This verifies the coordinator's mapping but does NOT verify that real Watch-detected cycles surface through the mapping. End-to-end coverage is deferred — see PR4.4 step 4's "End-to-end real-cycle coverage is deferred" note. If the user wants real-cycle coverage in Phase 1, the spec needs an additional decision: either expose `Derived::read` (Result-returning) inside compute closures as the canonical cycle-tolerant read for this test, or commit to a known incr-side recipe for constructing a cycle that propagates through `Watch::read` as `Err`.
