# moji ZWSP Sentinel Consolidation

## Why

The empty-paragraph sentinel codepoint U+200B (ZWSP) is hardcoded as a string
literal in six places, with no shared definition:

- MoonBit, insert: `lang/markdown/edits/compute_markdown_edit.mbt:202`
- MoonBit, strip-on-merge: `lang/markdown/edits/compute_markdown_edit.mbt:319`
- MoonBit, sentinel detect: `lang/markdown/companion/markdown_companion.mbt:81`
- TypeScript, display strip: `adapters/editor-adapter/block-input.ts:108,239,288`

Two consequences:

1. Any change to the sentinel codepoint (or its rules — e.g. "also treat a
   ZWNJ-only paragraph as empty") requires editing six sites with no compiler
   help.
2. The Unicode-property test ("is this string a visually-empty placeholder?")
   is duplicated rather than living in `lib/moji/`, even though moji is
   canopy's general Unicode library and already ships UCD-derived property
   tables.

The three-layer end state honors design principle 6 ("a framework's
generality is determined by what it excludes"):

- **moji** owns Unicode-general facts: the named codepoint constant
  `ZERO_WIDTH_SPACE` and the property predicate
  `is_default_ignorable_code_point`.
- **canopy `lang/markdown/sentinel/`** (a new canopy package, imported
  as `@md_sentinel`) owns the editor-policy role:
  `EMPTY_PARAGRAPH_SENTINEL` (a one-line alias of the moji constant) and
  `is_empty_paragraph_text` (the sentinel-detection predicate).
- **TypeScript** is not a parallel definition. The MoonBit FFI exports
  a `pub fn markdown_empty_paragraph_sentinel`; `examples/web` imports
  it from the `@moonbit/crdt-markdown` Vite virtual module and passes
  it into `BlockInput` as an option. The reusable
  `adapters/editor-adapter` package stays decoupled from any
  example-specific build wiring.

## Scope

In:
- `lib/moji/scripts/gen_property_tables.py` — add a new UCD table for
  `Default_Ignorable_Code_Point`.
- `lib/moji/property_tables.mbt` — regenerated table data.
- `lib/moji/property.mbt` — new public predicate.
- `lib/moji/codepoints.mbt` (new) — named codepoint constants surface,
  starting with `ZERO_WIDTH_SPACE`.
- `lib/moji/README.md` — document the inclusion rule for named codepoint
  constants (see Step 1b).
- `lib/moji/pkg.generated.mbti` — regenerated interface.
- `lang/markdown/sentinel/` (new package — **not a file in an existing
  package**; the existing `@markdown` is the grammar module sourced from
  `loom/examples/markdown`, not a canopy package). Imported as
  `@md_sentinel`. Owns `EMPTY_PARAGRAPH_SENTINEL` and
  `is_empty_paragraph_text`.
- `lang/markdown/companion/markdown_companion.mbt` — consume
  `@md_sentinel`.
- `lang/markdown/edits/compute_markdown_edit.mbt` — same.
- `ffi/markdown/markdown_ffi.mbt` — expose a `pub fn` accessor for the
  sentinel value.
- `ffi/markdown/moon.pkg` — add the new accessor to `link.js.exports`
  (the exports list is explicit; new symbols are not auto-exported).
- `adapters/editor-adapter/block-input.ts` — take a strip function as a
  `BlockInput` option rather than importing the FFI bundle directly
  (keeps the adapter package decoupled from Vite virtual modules).
- `examples/web/src/markdown-sentinels.ts` (new) — imports the FFI
  accessor via the `@moonbit/crdt-markdown` virtual module and wires it
  into the `BlockInput` option.

Out:
- Generalizing the sentinel rule to "default-ignorable-only paragraph"
  (recommended deferred — keep exact ZWSP match for now, add a doc-comment
  pointing at the moji predicate). See Risks.
- JSON5 lexer codepoints (U+200C, U+2028/9, U+FEFF) — those live in vendored
  `.mooncakes` and are out of scope.
- ZWJ handling — already properly consolidated inside moji's grapheme/word
  property tables.
- Test-fixture ZWSP literals in `integration_wbtest.mbt` — leave as-is; tests
  describing the sentinel behavior should keep the literal so the test reads
  as a behavioral assertion.
- Backfilling other named codepoint constants (ZWNJ, BOM, CGJ, …) beyond
  ZWSP. The inclusion rule we document in Step 1b governs future
  additions; do not add speculative constants in this plan.

## Current State

- `moji` ships 35 binary interval tables generated by
  `lib/moji/scripts/gen_property_tables.py` from four UCD inputs in
  `lib/moji/testdata/`. The script merges ranges, flattens to
  `[start0, end0, start1, end1, …]`, and writes `property_tables.mbt`.
  Lookup is binary search via `in_range_table(table, cp) -> Bool` in
  `property.mbt`.
- `DerivedCoreProperties.txt` is already loaded for InCB. Unicode 15.1
  ships **27** `Default_Ignorable_Code_Point` ranges in that file. InCB
  is parsed by a one-off manual block in the generator (not via
  `parse_property_file`) because `InCB` has three sub-values
  (`Consonant` / `Extend` / `Linker`) keyed off the *second* property
  column after filtering the first column to `InCB`, and the helper
  only returns a single property dimension per call.
  `Default_Ignorable_Code_Point` is a single-value property — the row
  format is `<range> ; Default_Ignorable_Code_Point` with no
  sub-value — so a clean
  `parse_property_file(dcp_path, {"Default_Ignorable_Code_Point"}, prop_index=0)`
  call works without modification. See Step 1a Item 1 for the concrete
  change. Verified: the helper signature
  `parse_property_file(path, accepted_props, prop_index=0)` is already
  in the generator (`lib/moji/scripts/gen_property_tables.py:27`).
- Adding a new predicate is one entry in `table_specs` + one `pub fn` in
  `property.mbt`. The table-generator pattern already handles "extract one
  named property and emit a `Bool` lookup." See the `incb_consonant` /
  `extended_pictographic` precedents.
- moji currently exposes property *enums* and *predicates* but no named
  codepoint *constants*. This plan introduces that category; the inclusion
  rule (Step 1b) prevents it from growing opportunistically.
- On the canopy side, `markdown_companion.mbt::is_zwsp_sentinel` is the
  authoritative sentinel-detection function and is itself called from
  `collect_zwsp_ranges`. Other MoonBit sites are smaller inline checks.
- The TS sites are all `.replace(/​/g, '')` regex strips inside the
  reusable `adapters/editor-adapter` package. They do not care about full
  Unicode-property semantics; they care about one specific codepoint.
- `@markdown` in canopy's `lang/markdown/{companion,edits}` packages
  resolves to `dowdiness/markdown`, the **grammar package** from
  `loom/examples/markdown` (a submodule). There is no canopy-owned
  `lang/markdown/` package. The role-name layer must live in a new
  canopy package (this plan creates `lang/markdown/sentinel/`).
- `ffi/markdown/moon.pkg` lists JS exports explicitly in `link.js.exports`.
  A new `pub fn` is not auto-exposed to the bundle; the symbol name must
  be added to that list.
- The reusable adapter `adapters/editor-adapter` does **not** import any
  FFI bundle directly; consumers like `examples/web` import the
  `@moonbit/crdt-markdown` Vite virtual module (defined in
  `examples/web/vite.config.ts`) and wire FFI calls into the adapter
  through constructor options.

## Desired State

- moji exposes:
  - `pub const ZERO_WIDTH_SPACE : String` (the U+200B literal).
  - `pub fn is_default_ignorable_code_point(cp : Int) -> Bool` with UCD
    15.1 coverage and a unit test against a representative sample (CGJ,
    ZWSP, ZWNJ, BOM, language tags).
  - A documented inclusion rule in `lib/moji/README.md` governing which
    codepoints earn a named constant.
- `lang/markdown/sentinel/` is a new canopy package (`@md_sentinel`)
  owning:
  - `pub const EMPTY_PARAGRAPH_SENTINEL : String = @moji.ZERO_WIDTH_SPACE`.
  - `pub fn is_empty_paragraph_text(s : String) -> Bool` (exact match for
    now; doc-comment notes the moji predicate is available if the rule
    needs to broaden).
- Every MoonBit call site that inserts, detects, or strips the sentinel
  goes through `@md_sentinel`.
- `ffi/markdown/markdown_ffi.mbt` defines
  `pub fn markdown_empty_paragraph_sentinel() -> String` returning
  `@md_sentinel.EMPTY_PARAGRAPH_SENTINEL`. The symbol is listed in
  `ffi/markdown/moon.pkg` `link.js.exports` and is reachable in
  `markdown.d.ts` after `moon build --target js --release`.
- The reusable adapter exposes a strip function as a `BlockInput` option:
  ```ts
  new BlockInput({ stripParagraphSentinels: (s) => /* ... */ })
  ```
  with a permissive default that strips ZWSP for backwards compatibility
  if not passed.
- `examples/web/src/markdown-sentinels.ts` imports the FFI accessor from
  `@moonbit/crdt-markdown`, captures the value once, and constructs the
  `stripParagraphSentinels` function passed into `BlockInput`. No
  literal ZWSP in this file beyond the `import { ... }` line; the value
  flows from the MoonBit build artifact.
- `git grep` (covering all escape forms and the literal codepoint) over
  `'*.mbt' '*.ts' '*.tsx'` returns only `lib/moji/codepoints.mbt`, moji's
  property-predicate tests, and annotated test fixtures in
  `lang/markdown/companion/integration_wbtest.mbt`.

## Steps

### Step 1a — moji `Default_Ignorable_Code_Point` predicate

Add the property to moji's generator and public API.

1. Edit `lib/moji/scripts/gen_property_tables.py`:
   - Add a new parse pass beside the existing four
     (`gcb`, `wb`, `emoji`, `incb`):
     `dcp = parse_property_file(dcp_path, {"Default_Ignorable_Code_Point"}, prop_index=0)`.
     Do **not** reuse the InCB manual block — that block exists because
     InCB has three sub-values keyed off column 2 after filtering
     column 1 = "InCB". `Default_Ignorable_Code_Point` is a flat
     single-value property and the standard helper handles it.
   - Append a `table_default_ignorable_code_point` entry to the
     `table_specs` list around line 162 of the generator.
2. Run `python3 lib/moji/scripts/gen_property_tables.py` to regenerate
   `property_tables.mbt`. Sanity-check the emitted table has 27 ranges
   (Unicode 15.1 count).
3. Add `pub fn is_default_ignorable_code_point(cp : Int) -> Bool` to
   `lib/moji/property.mbt`, body is one `in_range_table` call.
4. Add a `*_wbtest.mbt` case asserting representative codepoints
   (U+034F CGJ, U+200B ZWSP, U+200C ZWNJ, U+200E LRM, U+FEFF BOM,
   U+E0001 LANGUAGE TAG) return `true`, and that ASCII/CJK letters return
   `false`.

### Step 1b — moji named codepoint constants surface + inclusion rule

Introduce the new "named codepoint constants" API category with the
narrowest possible footprint and a documented rule that won't be gamed
by adding more conformance test data.

1. Add `lib/moji/codepoints.mbt` containing:
   ```moonbit
   /// U+200B ZERO WIDTH SPACE. Exposed as a named constant because the
   /// markdown editor's empty-paragraph sentinel role (see
   /// `lang/markdown/sentinel/`) consumes this specific scalar by name.
   pub const ZERO_WIDTH_SPACE : String = "\u{200B}"
   ```
2. Update `lib/moji/README.md` with a short subsection — proposed title
   "Named codepoint constants" — stating the inclusion rule:

   > moji ships a named `pub const` for a codepoint only when at least
   > one of the following holds:
   >
   > 1. **External role.** A current non-moji consumer in the canopy
   >    workspace needs that exact scalar by role or name (not by
   >    property class). The consumer's call site is named in the
   >    constant's docstring.
   > 2. **Public-API canonical value.** moji's own public API contract
   >    exposes that scalar as a canonical value (e.g. as the documented
   >    return value of a function, or as the conventional sentinel for
   >    a result).
   >
   > Appearing in moji's *internal* tests, conformance fixtures, or
   > property-predicate test inputs does **not** qualify. Adding a
   > constant outside these criteria requires opening an issue first.

   `ZERO_WIDTH_SPACE` qualifies under rule 1: `@md_sentinel`
   (`lang/markdown/sentinel/`) consumes it by role.
   Codepoints that appear only as conformance inputs (CGJ, ZWNJ, LRM,
   BOM, LANGUAGE TAG in `is_default_ignorable_code_point`'s tests) do
   **not** qualify under either rule and must remain inline literals
   in their test files.
3. Do **not** experiment with `pub const String` JS-binding exposure.
   The plan exports the value to TS via a small FFI accessor function
   (Step 4) regardless of whether the const is directly importable —
   this keeps the FFI surface uniform and removes a branch from the
   plan. The const stays useful inside MoonBit; the JS side always goes
   through the function.

### Step 2 — Create the canopy package `lang/markdown/sentinel/`

`@markdown` in canopy's `lang/markdown/{companion,edits}` resolves to
the grammar package `dowdiness/markdown` from
`loom/examples/markdown`. There is no canopy-owned `@markdown` package
to add a file to, and putting editor-policy code into a vendored
grammar submodule would invert the dependency direction.

Create a new canopy package `dowdiness/canopy/lang/markdown/sentinel`,
imported as `@md_sentinel`.

1. Add `lang/markdown/sentinel/moon.pkg` declaring the package and an
   import on `dowdiness/moji`. Follow the existing
   `lang/markdown/companion/moon.pkg` style for shape.
2. Add `lang/markdown/sentinel/sentinel.mbt`:
   ```moonbit
   /// Role-name layer for the empty-paragraph sentinel codepoint.
   ///
   /// The literal codepoint identity lives in `@moji.ZERO_WIDTH_SPACE`.
   /// This module names the editor-policy role: "the codepoint we
   /// insert to keep an empty paragraph visible and editable."

   pub const EMPTY_PARAGRAPH_SENTINEL : String = @moji.ZERO_WIDTH_SPACE

   /// True when the entire string is exactly the sentinel codepoint.
   ///
   /// Intentionally narrow today: legitimate user text containing the
   /// sentinel character alongside other content returns false. To
   /// broaden the rule to "any default-ignorable-only paragraph"
   /// (catching paste of ZWNJ, BOM, etc.), iterate codepoints and
   /// consult `@moji.is_default_ignorable_code_point`.
   pub fn is_empty_paragraph_text(s : String) -> Bool {
     s == EMPTY_PARAGRAPH_SENTINEL
   }
   ```
3. Add a `sentinel_wbtest.mbt` covering: exact match → true; empty
   string → false; two sentinels → false; sentinel + "a" → false.
4. Run `moon check && moon test && moon info && moon fmt` from repo
   root (the workspace covers this new package).

### Step 5 — TypeScript wiring (adapter option + example glue)

The reusable adapter at `adapters/editor-adapter` must stay decoupled
from any specific consumer's build wiring (Vite virtual modules,
bundler aliases, FFI paths). The FFI import happens in the example
that owns its build config; the adapter receives the strip function as
an option.

1. **In `adapters/editor-adapter/block-input.ts`:**
   - Today the class declares
     `constructor(container: HTMLElement)`. Extend it to
     `constructor(container: HTMLElement, opts: BlockInputOptions = {})`.
     The defaulted second argument keeps all existing callers
     (`new BlockInput(container)`) compiling unchanged.
   - Add the options interface:
     ```ts
     export interface BlockInputOptions {
       /** If provided, used to strip empty-paragraph sentinel codepoints
        *  from display text. Defaults to a permissive ZWSP-only strip
        *  for backwards compatibility with consumers that haven't wired
        *  the FFI-sourced function. */
       stripParagraphSentinels?: (s: string) => string
     }
     ```
   - Store the resolved strip function (with the fallback applied) on
     `this`:
     ```ts
     private stripParagraphSentinels: (s: string) => string
     // ... in the constructor:
     this.stripParagraphSentinels =
       opts.stripParagraphSentinels ?? ((s) => s.replace(/​/g, ''))
     ```
   - Replace the three inline `.replace(/​/g, '')` sites (lines
     108, 239, 288) with `this.stripParagraphSentinels(...)`.
2. **Add `examples/web/src/markdown-sentinels.ts`:**
   ```ts
   import { markdown_empty_paragraph_sentinel } from "@moonbit/crdt-markdown"

   /** Captured once at module init. Source: MoonBit FFI bundle ←
    *  @md_sentinel.EMPTY_PARAGRAPH_SENTINEL ← @moji.ZERO_WIDTH_SPACE. */
   const SENTINEL = markdown_empty_paragraph_sentinel()

   export function stripParagraphSentinels(s: string): string {
     return s.split(SENTINEL).join("")
   }
   ```
3. **In `examples/web/src/markdown-editor.ts`** (or wherever
   `BlockInput` is constructed for the markdown editor): pass
   `stripParagraphSentinels` from the new module into the
   `BlockInput` options.
4. Run `cd examples/web && npx tsc --noEmit` after `moon build --target js --release`
   has populated `_build/js/release/...`.

## Risks

- **Generalization scope creep.** The current sentinel rule is exact-match
  ZWSP. Broadening to "default-ignorable-only paragraph" would catch user
  pastes of ZWNJ/BOM but might also misclassify edge content. Defer this
  decision; keep exact match and a doc-comment hook into moji.
- **Inclusion rule erosion.** Once moji ships `ZERO_WIDTH_SPACE`, future
  contributors may add `ZERO_WIDTH_JOINER`, `BOM`, `CGJ`, etc.
  opportunistically. The rule in `lib/moji/README.md` is the line of
  defense, including the explicit "appears only in conformance tests
  does NOT qualify" clause; reviewers should reject additions that
  don't cite the rule. If the rule needs to evolve (e.g. once a second
  external consumer wants ZWJ as a constant), that's a separate API
  decision worth its own plan.
- **Build-order fragility.** TS now imports a value computed by the
  MoonBit JS-release build. If `moon build --target js --release` is
  not run before `npx tsc --noEmit` in a fresh checkout, the import
  will fail with a module-not-found error. `examples/web` already
  depends on the FFI bundle being built (existing imports of
  `markdown_apply_edit` etc.), so adding one more imported symbol does
  not change the build prerequisites. Confirm during execution.
- **Adapter option migration.** Adding a `BlockInput` option is a
  non-breaking change because the option has a default. Consumers
  using the adapter outside `examples/web` keep working with the
  fallback strip. Document the option in the adapter README so the
  FFI-sourced wiring is the obvious choice for new consumers.
- **Performance.** `is_default_ignorable_code_point` adds one 27-range
  binary-search table to moji's public surface. Memory cost is
  negligible (~220 bytes). Lookup is O(log n) like every other moji
  predicate.
