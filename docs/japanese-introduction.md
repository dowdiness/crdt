# Canopy

コードを書いているとき、あなたの頭の中にはプログラムの「意味」がある。この変数が何を指しているか、この関数が何を返すか、このコードがどう動くか。

でもエディタはそれを何も知らない。ただの文字列としてコードを扱っている。

Canopyは、書いた瞬間からプログラムの意味を追いかけるエディタだ。構文木がリアルタイムに構築され、変数のスコープが解決され、式の評価結果が表示され、コードが自動的に整形される——すべてが、1文字タイプするたびに更新される。

テキストと構文木は双方向に同期する。テキストを編集すれば構造が追従し、構造を操作すればテキストが書き換わる。複数人での同時編集もサーバー不要で動作する。

特定の言語に縛られていない。文法を与えれば、この体験がそのまま手に入る。

## もう少し詳しく

### あなたが毎日やっている翻訳

コードを書くとき、あなたは常に翻訳をしている。

頭の中には「この関数は引数を2倍にする」という理解がある。でもそれをエディタに伝えるには、`λx. x + x` という文字の並びに変換しなければならない。リファクタリングするときは、コードを目で読んで頭の中に意味を復元し、変更を考え、また文字列に変換して書き戻す。

この翻訳を、あなたは一日に何百回も繰り返している。

バグを探すときも同じだ。コードをじっと見て、頭の中で実行をシミュレートし、「ここでこの変数はこの値になるはず」と推論する。エディタはその推論を一切手伝わない。変数の値も、型も、スコープも、あなたの頭の中にしかない。

この翻訳コストは、プログラミングそのものの一部だと思われている。でも本当は、道具の限界がもたらしているものだ。ロジックを考えることは難しくない。それを何も理解していないテキストエディタを経由して表現することが、苦しいのだ。

### 構造が「生まれる」とはどういうことか

Canopyで文字をタイプすると、その瞬間にパーサーが動く。ただし、ファイル全体を解析し直すのではない。変更の影響を受ける部分だけが再解析され、残りは前回の結果がそのまま使われる。

この部分的な再解析から、構文木が更新される。変数のスコープが解決される。式の型が推論される。評価可能な式は、その場で結果が計算される。

あなたは何もしていない。ただ書いただけだ。構造は、書くという行為の副産物として自然に立ち上がる。

これが「自動的に構造が生まれる」の意味だ。フォーマットボタンを押すのでも、「解析」を実行するのでもない。書くことと構造化が、同じ一つの動作になっている。

### テキストからも、構造からも

普通のエディタでは、コードに触れる手段はテキスト編集だけだ。関数の引数を入れ替えたければ、文字列をカット＆ペーストする。式を関数に抽出したければ、自分でテキストを切り出して新しい関数を書き、元の場所を呼び出しに書き換える。

Canopyでは、構文木を直接操作できる。テキストとして編集してもいいし、構造として操作してもいい。どちらで変更しても、もう一方が自動的に追従する。

構造を操作するということは、意味のレベルで編集するということだ。「この引数を入れ替える」はテキスト上の操作ではなく、構文木上の操作になる。テキストの帳尻合わせ——括弧の対応、インデントの調整、参照の更新——はエディタが引き受ける。

あなたは「何をしたいか」だけを伝えればいい。

### コードの先にあるもの

Canopyがコードエディタとして解いている問題は、実はコードに限った話ではない。

「テキストから自動的に構造を抽出する」「関連性を追跡する」「必要な情報を必要なタイミングで表示する」——これらはすべて、あらゆる「書く」行為に共通する課題だ。

メモを書く。アイデアを記録する。URLを保存する。読んだ本の感想を残す。こうした断片的な入力に対しても、同じ原理が適用できる。書いた瞬間から構造が生まれ、断片の間に関連性が発見され、何かを書き始めたときに過去の文脈が自動的に浮かび上がる。

Canopyは、この体験を実現するための技術的基盤をコードエディタの中で鍛えている。テキストからの自動構造化。意味の追跡と増分更新。サーバー不要の複数デバイス同期。これらの歯車がすべてかみ合うことを、コードという最も構造が明確な世界で証明する。その先に、あらゆる種類の「書くこと」に同じ体験を広げる道がある。

### 距離を縮める

プログラミングが難しいのは、あなたの意図とコンピュータの理解の間に距離があるからだ。

あなたは「こうしたい」と思っている。でもそれを伝えるために、意味を構造に、構造を構文に、構文を文字の列に——何段階もの変換を経なければならない。

Canopyは、この距離を一段ずつ縮めていく。文字列を構造にする。構造に意味を付ける。意味をあなたの目に見える形にする。

道具を使っていることを意識しなくなる瞬間がある。手に持ったペンの存在を忘れて、ただ書いている。楽器を弾いていて、指の動きではなく音楽そのものに没入している。

コンピュータでその瞬間を作ること——考えたことがそのまま形になり、意図と表現の間に翻訳がいらなくなること。それがCanopyの目指すところだ。
